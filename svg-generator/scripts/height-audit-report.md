# Height Audit Report

Generated by `pnpm audit:heights` (full run: 10 plugins, ~58 sections × {half,full} × {default,terminal} = 232 measurements).

**Summary: 232 tested | 13 ok | 150 overflow | 69 oversized | 0 errors**

- **overflow** = `calculateHeight()` returned LESS than the real rendered height → content can get cut off in the generated SVG (violates the "safe upper-bound" contract).
- **oversized** = `calculateHeight()` returned more than 20px MORE than the real rendered height → wasted blank space in the SVG.

## Tier 2 — Systemic: `calculateHeight()` doesn't account for `style` (terminal)

`calculateHeight(config, data, size)` returns the **same number** for `default` and `terminal`, but the terminal layout for list/grid-style sections renders **dramatically differently** — in both directions. This is one root cause affecting ~20 sections across nearly every plugin. Likely a shared terminal "list/grid" component issue (possible `overflow:hidden`/fixed-height clipping in terminal mode for the "oversized" cases — i.e. terminal might be **silently dropping list items**, not just wasting space).

## Recommended approach

1. **Tier 2 first** — it's one architectural gap (`calculateHeight` signature lacks `style`) with the broadest blast radius (~20 sections, nearly every plugin). Worth checking visually whether the "terminal renders much less" cases are actual content clipping (bug) vs. intentional compact terminal layouts (then `calculateHeight` just needs a terminal-specific formula).
2. **Tier 1 next** — fix the biggest default-style overflows (github/recent_activity, myanimelist favorites sections, github/stargazers, github/calendar, github/repository_contributors) since these affect the most common style.
3. **Tier 3 last** — minor, just wasted space in `full` size.

Re-run with filters during fixes, e.g.:
```bash
pnpm audit:heights -- --plugin=github --section=recent_activity
pnpm audit:heights -- --plugin=myanimelist
```
